## 🔧 优化方案

### 方案 1：后端直接 HTTP 调用（核心优化）⭐⭐⭐⭐⭐

#### 问题诊断
**原方案**: 使用 LangChain 的 `astream()` 方法
```python
# ❌ 伪流式 - 实际是批量输出
async for chunk in await llm.astream(messages):
    yield chunk.content
```

**问题**:
- LangChain 内部会等待模型生成一定长度的 chunk 才输出
- 不是真正的实时流式
- 首字延迟高达 27 秒

#### 优化方案
**新方案**: 直接使用 HTTP 调用 API
```python
# ✅ 真正的 SSE 流式
async with httpx.AsyncClient(timeout=None) as client:
    response = await client.post(
        f"{settings.GUIJI_API_BASE}/chat/completions",
        headers={"Authorization": f"Bearer {settings.GUIJI_API_KEY}"},
        json={
            "model": settings.GUIJI_MODEL,
            "messages": messages,
            "stream": True,
        },
    )
    
    async for line in response.aiter_lines():
        if line.startswith("data: "):
            data = json.loads(line[6:])
            content = data["choices"][0]["delta"]["content"]
            yield content  # 立即输出
```

**效果**:
- ✅ 真正的实时流式
- ✅ 首字延迟降低到 14 秒
- ✅ 代码完全可控

#### 实现细节
```python
async def process_question_upload_stream(
    self,
    user_id: int,
    question_id: int,
    image_contents: List[bytes],
    description: Optional[str] = None
) -> AsyncGenerator[str, None]:
    """处理题目上传 - 流式输出版（直接 HTTP 调用 API）"""
    
    # 1. 构建多模态提示
    content = []
    
    # 添加图片（转换为 base64）
    for image_content in image_contents:
        base64_image = base64.b64encode(image_content).decode()
        content.append({
            "type": "image_url",
            "image_url": {"url": f"data:image/png;base64,{base64_image}"}
        })
    
    # 构建文本指令
    text_instruction = self._build_analysis_prompt(description)
    content.append({
        "type": "text",
        "text": text_instruction
    })
    
    messages = [{"role": "user", "content": content}]
    
    # 2. 直接调用 API（真正的流式）
    async with httpx.AsyncClient(timeout=None) as client:
        response = await client.post(
            f"{settings.GUIJI_API_BASE}/chat/completions",
            headers={
                "Authorization": f"Bearer {settings.GUIJI_API_KEY}",
                "Content-Type": "application/json",
            },
            json={
                "model": settings.GUIJI_MODEL,
                "messages": messages,
                "stream": True,
                "max_tokens": settings.GUIJI_MAX_TOKENS,
                "temperature": settings.GUIJI_TEMPERATURE,
            },
        )
        
        response.raise_for_status()
        
        # 3. 流式解析 SSE 响应
        async for line in response.aiter_lines():
            if line.startswith("data: "):
                data_str = line[6:]
                if data_str == "[DONE]":
                    break
                
                try:
                    data = json.loads(data_str)
                    choices = data.get("choices", [])
                    if choices:
                        delta = choices[0].get("delta", {})
                        content_delta = delta.get("content", "")
                        if content_delta:
                            yield content_delta  # 立即输出
                except json.JSONDecodeError:
                    continue
```

### 方案 3：前端图片压缩 ⭐⭐⭐⭐⭐

#### 问题诊断
- 原图大小：~14KB
- 传输时间：~2 秒
- 增加不必要的延迟

#### 优化方案
**工具**: `expo-image-picker`
```typescript
// mobile/src/utils/imageCompressor.ts
import * as ImagePicker from 'expo-image-picker';

export const takePhotoAndCompress = async (
  options: {
    maxWidth?: number;
    maxHeight?: number;
    quality?: number;
  } = {}
): Promise<string | null> => {
  const {
    maxWidth = 800,
    maxHeight = 600,
    quality = 0.7,
  } = options;

  // 拍照并自动压缩
  const result = await ImagePicker.launchCameraAsync({
    mediaTypes: ImagePicker.MediaTypeOptions.Images,
    allowsEditing: true,
    aspect: [4, 3],
    quality: quality,  // 70% 质量
    base64: false,
  });

  if (!result.canceled && result.assets && result.assets.length > 0) {
    return result.assets[0].uri;
  }

  return null;
};
```

**效果**:
- 图片大小：14KB → **~5KB** ⬇️ 64%
- 传输时间：2 秒 → **0.5 秒** ⬇️ 75%
- 首字延迟：再降低 1-2 秒

#### 使用示例

from fastapi.responses import StreamingResponse

@app.post("/api/v1/questions/upload/stream")
async def upload_question_stream(...):
    async def generate():
        async for chunk in solver.process_question_upload_stream(...):

### 前端流式 Hook
```typescript
// mobile/src/hooks/useStreaming.ts
export const useStreaming = () => {
  const [data, setData] = useState<string>('');
  const [loading, setLoading] = useState(false);

  const connect = useCallback((url: string) => {
    const eventSource = new EventSource(url);
    
    eventSource.onmessage = (event) => {
      const parsed = JSON.parse(event.data);
      if (parsed.chunk) {
        setData((prev) => prev + parsed.chunk);
      }
    };
  }, []);

  return { data, loading, connect };
};
```

## ⚠️ 踩坑记录

### 坑 1: LangChain 伪流式
**现象**: 使用 `astream()` 但不是真正的实时输出  
**原因**: LangChain 内部会缓冲一定长度才输出  
**解决**: 直接 HTTP 调用 API

### 坑 2: 数据库连接关闭
**现象**: 流式输出中途数据库连接关闭  
**原因**: 会话超时  
**解决**: 在流式生成前完成数据库操作

### 坑 3: SSE 格式错误
**现象**: 前端无法解析响应  
**原因**: SSE 格式不规范  
**解决**: 严格遵循 `data: {...}\n\n` 格式

### 坑 4: 图片上传超时
**现象**: 大图片上传超时  
**原因**: 默认超时时间太短  
**解决**: 设置 `timeout=None` 或增加超时时间
