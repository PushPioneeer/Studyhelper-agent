# 学习助手 - 系统架构设计

## 1. 系统总体架构

### 1.1 架构概述

本系统采用**分层架构 + 微服务预留**的设计，整体分为四层：

```
┌─────────────────────────────────────────────────────────┐
│                    客户端层 (Client)                      │
│              React Native 移动端应用                       │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                  BFF 层 (Backend for Frontend)            │
│              Node.js + Fastify                           │
│     · API 网关    · 认证授权    · 请求聚合    · 限流      │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                  应用服务层 (Application)                 │
│              FastAPI + LangChain Agents                  │
│  · 解题服务  · 课件解析服务  · 错题服务  · 用户服务       │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                  数据层 (Data & Infrastructure)          │
│    PostgreSQL    ·    Redis    ·    本地文件存储         │
└─────────────────────────────────────────────────────────┘
```

### 1.2 架构特点

- **前后端分离**: 移动端与后端通过 RESTful API 通信
- **BFF 模式**: Node.js 层作为移动端专属后端，负责协议转换和请求聚合
- **服务解耦**: FastAPI 各功能模块独立，便于后期微服务化
- **缓存优化**: Redis 缓存热点数据，提升响应速度
- **异步处理**: 耗时操作异步执行，提升用户体验

```python
        )
    
    @tool
    def analyze_question(self, image_url: str) -> str:
        """识别并分析题目图片内容。
        
        Args:
            image_url: 题目图片的 URL 或 base64 编码
        
        Returns:
            识别出的题目文本和分析结果

                {"type": "image_url", "image_url": {"url": image_url}}

        response = self.llm.invoke(messages)

        Returns:

            {"role": "user", "content": f"请解答这道题目并给出详细步骤，使用 Markdown 格式排版：{question_text}"}
        ]
        response = self.llm.invoke(messages)
        return response.content

        ]

        return self._parse_knowledge_points(response.content)
    
    @tool

        Args:
            knowledge_points: 知识点列表
```

**图片编码工具函数**
```python
import requests
import base64
import mimetypes

def encode_image(image_path: str) -> str:
    """
    将图片编码为 base64 字符串，供硅基流动 API 使用
    
    Args:
        image_path: 图片路径（本地文件路径或网络 URL）
    
    Returns:
        base64 编码的图片字符串，格式：data:image/jpeg;base64,{base64_data}
    """
    if image_path.startswith("http://") or image_path.startswith("https://"):
        # 下载网络图片
        response = requests.get(image_path)
        if response.status_code == 200:
            image_content = response.content
        else:
            raise Exception(f"下载图片失败：{response.status_code}")
        # 猜测 MIME 类型
        mime_type, _ = mimetypes.guess_type(image_path)
        if mime_type is None:
            mime_type = "application/octet-stream"
    else:
        # 读取本地文件
        with open(image_path, "rb") as f:
            image_content = f.read()
        mime_type, _ = mimetypes.guess_type(image_path)
        if mime_type is None:
            mime_type = "application/octet-stream"
    
    # 返回 base64 编码
    base64_data = base64.b64encode(image_content).decode()
    return f"data:{mime_type};base64,{base64_data}"
```

```python
# 移动端 Agent 会话配置
class AgentSession:
    """移动端 Agent 会话管理器"""
    
    async def invoke_agent(self, user_id: str, question_id: str, message: str):
        """调用 Agent 处理用户请求"""
        agent = self.question_agent.create_agent()
        
        # 移动端会话配置：thread_id 关联用户和题目
        config = {
            "configurable": {
                "thread_id": f"{user_id}:{question_id}"
            },
            "recursion_limit": 15  # 防止无限循环
        }
        
        # 调用 Agent
        result = await agent.ainvoke({
            "messages": [{"role": "user", "content": message}]
        }, config=config)
        
        # 获取最后一条消息（Agent 回复）
        response = result["messages"][-1].content
        
        return response
    
    async def check_follow_up_limit(self, user_id: str, question_id: str) -> bool:
        """检查追问次数（移动端硬性限制：10 次）"""
        # 从数据库查询该用户对该题目的追问次数
        count = await db.query(FollowUp).filter(
            FollowUp.user_id == user_id,
            FollowUp.question_id == question_id
        ).count()
        
        return count < 10  # 硬性限制：10 次
```

**缓存实现**
```python
class CacheService:
    def __init__(self):
        self.redis = Redis(
            host=settings.REDIS_HOST,
            port=settings.REDIS_PORT,
            db=0,
            decode_responses=True,
        )
    
    async def get(self, key: str) -> Optional[Any]:
        data = await self.redis.get(key)
        return json.loads(data) if data else None
    
    async def set(self, key: str, value: Any, ttl: int = 3600):
        await self.redis.setex(key, ttl, json.dumps(value, ensure_ascii=False))
    
    async def delete(self, key: str):
        await self.redis.delete(key)
    
    async def increment(self, key: str, ttl: int = 3600) -> int:
        pipe = self.redis.pipeline()
        pipe.incr(key)
        pipe.expire(key, ttl)
        await pipe.execute()
        return int(await self.redis.get(key))
```

```python
class FileStorageService:
    BASE_DIR = Path("./storage")
    
    async def save_image(self, file: UploadFile, user_id: str) -> str:
        """保存图片"""
        filename = f"{uuid.uuid4()}.jpg"
        path = self.BASE_DIR / "images" / str(user_id) / filename
        path.parent.mkdir(parents=True, exist_ok=True)
        
        # 保存原图
        with open(path, "wb") as f:
            f.write(await file.read())
        
        # 生成压缩图
        compressed_path = path.parent / f"compressed_{filename}"
        await self.compress_image(path, compressed_path)
        
        return str(path)
    
    async def save_courseware(self, file: UploadFile, user_id: str) -> str:
        """保存课件"""
        filename = f"{uuid.uuid4()}_{file.filename}"
        path = self.BASE_DIR / "courseware" / str(user_id) / filename
        path.parent.mkdir(parents=True, exist_ok=True)
        
        with open(path, "wb") as f:
            f.write(await file.read())
        
        return str(path)
```

## 5. 性能优化策略

### 5.1 缓存优化
- **多级缓存**: 客户端缓存 → Redis 缓存 → 数据库
- **缓存预热**: 热门知识点、常用数据预加载
- **缓存失效**: 数据更新时主动失效相关缓存

### 5.2 数据库优化
- **连接池**: 最大 50 连接，避免频繁创建连接
- **索引优化**: 常用查询字段建立索引
- **分页优化**: 使用游标分页，避免 OFFSET 过大
- **慢查询监控**: 记录并优化慢查询

### 5.3 异步处理
- **耗时操作异步化**: 课件解析、笔记生成
- **任务队列**: Redis + Celery
- **进度查询**: WebSocket 或轮询
