# 学习助手 - 需求分析文档

## 1. 项目概述

### 1.1 项目名称
学习助手 (Study Helper)

### 1.2 项目定位
一款面向大学生的智能学习辅助移动应用，通过 AI 技术提供题目识别、解题讲解、知识点梳理、课件解析、错题管理等功能，帮助学生提升学习效率。

### 1.3 目标用户
- 主要用户：大学生群体
- 次要用户：高中生、自学者

### 1.4 核心价值
- 📸 **拍照解题**：快速识别题目并提供详细解析
- 💬 **追问互动**：支持对话式深度学习
- 📚 **课件解析**：自动生成精美学习笔记
- 📝 **错题管理**：智能收录和整理错题
- 💾 **学习记忆**：完整记录学习轨迹

---

## 2. 功能需求

### 2.1 用户系统

#### 2.1.1 功能描述
提供用户注册、登录、个人信息管理功能。

#### 2.1.2 详细需求
- **手机号注册/登录**
  - 支持手机号 + 验证码登录
  - 支持手机号 + 密码登录
  - 验证码有效期：5 分钟
  - 同一手机号 60 秒内只能获取一次验证码
  
- **用户信息管理**
  - 昵称（必填）
  - 头像（可选）
  - 学校/专业（可选）
  - 学习偏好设置（可选）

- **登录状态管理**
  - JWT Token 认证
  - Token 有效期：7 天
  - Refresh Token 有效期：30 天
  - 支持多设备登录

#### 2.1.3 业务规则
- 每个手机号只能注册一个账号
- 支持账号注销
- 支持修改绑定手机号

---

### 2.2 拍照解题功能

#### 2.2.1 功能描述
用户拍照或上传图片，系统识别题目并提供详细解析。

#### 2.2.2 详细需求

**图片上传**
- 支持拍照上传
- 支持从相册选择
- 支持多张图片（最多 5 张）
- 图片格式：JPG、PNG
- 单张图片大小限制：10MB
- 图片压缩：前端压缩至 2MB 以内

**题目识别**
- 使用硅基流动 GLM-4.1V-9B-Thinking 模型
- 支持印刷体题目识别
- 支持手写体题目识别（清晰度要求较高）
- 支持数学公式、化学方程式等特殊格式
- 识别准确率目标：>90%

**题目讲解**
- 详细解题步骤
- 关键知识点标注
- 支持多种解法展示
- 解题思路引导

**知识点清单**
- 提取题目涉及的核心知识点
- 知识点难度分级（基础/进阶/提高）
- 关联知识点推荐
- 知识点掌握程度评估

**类似题目生成**
- 基于原题知识点生成类似题目
- 支持点击生成
- 每次生成 3-5 道
- 难度可调节（简单/中等/困难）

#### 2.2.3 追问功能
- **追问模式**：对话式交互
- **追问次数限制**：每道题目最多 10 次
- **追问内容**：
  - 步骤详解
  - 概念澄清
  - 变式提问
  - 拓展延伸
- **追问记录**：保存完整的对话历史
- **次数用完处理**：硬性限制，不可继续追问

#### 2.2.4 业务规则
- 每道题目独立计算追问次数
- 追问次数不重置、不累计
- 不支持删除追问记录（保证学习完整性）

---

### 2.3 课件解析功能

#### 2.3.1 功能描述
上传课件文件，系统解析内容并生成详细学习笔记。

#### 2.3.2 详细需求

**支持格式**
- PDF (.pdf)
- PPT/PPTX (.ppt, .pptx)
- Word (.doc, .docx)
- Markdown (.md)

**文件限制**
- 单文件大小：≤50MB
- PPT 页数：≤200 页
- PDF 页数：≤500 页
- Word 文档：≤100 页

**内容解析**
- 文本提取
- 图片识别（图表、公式）
- 结构分析（章节、段落）
- 重点内容标记

**笔记生成要求**
- ❌ 不要大纲式内容
- ✅ 详细的知识点讲解
- ✅ 概念解释 + 示例
- ✅ 重点难点标注
- ✅ 知识关联和拓展
- ✅ 排版精美、层次清晰

**笔记结构**
```markdown
# 课件标题

## 一、核心概念
### 1.1 概念名称
- 定义：...
- 特点：...
- 示例：...
- 应用场景：...

## 二、重要原理
### 2.1 原理名称
- 原理描述：...
- 推导过程：...
- 注意事项：...

## 三、典型例题
### 3.1 例题 1
- 题目：...
- 解析：...
- 答案：...

## 四、知识总结
- 重点回顾
- 易错点提醒
- 学习建议
```

#### 2.3.3 业务规则
- 课件解析需要一定时间，采用异步处理
- 支持解析进度查询
- 解析失败支持重新上传
- 笔记支持导出（PDF、Markdown）

---

### 2.4 错题集功能

#### 2.4.1 功能描述
自动收录错题，支持整理、复习、生成练习。

#### 2.4.2 详细需求

**错题收录**
- **自动收录**：解题过程中识别为错误的题目自动加入
- **手动添加**：用户可手动添加错题
- **收录内容**：
  - 原题图片
  - 题目文本
  - 正确答案
  - 用户答案
  - 解析内容
  - 知识点标签
  - 错题时间
  - 错题来源（拍照/练习）

**错题分类**
- 按学科分类（数学、物理、化学、英语等）
- 按知识点分类
- 按错误类型分类（概念不清/计算错误/理解偏差）
- 按难度分类
- 自定义标签

**错题整理**
- 支持批量选择
- 支持删除错题
- 支持移动分类
- 支持添加备注
- 支持标记掌握程度

**格式示范**（预留接口）
```
【待实现】具体格式由用户后期提供
```

**生成练习**
- 支持选择多道错题
- 根据错题生成类似例题
- 支持组卷练习
- 支持答案解析

#### 2.4.3 业务规则
- 错题收录后不可删除原题（保证学习记录完整）
- 支持"已掌握"标记，标记后移入已掌握题库
- 错题可导出（PDF、Word）

## 3. 性能需求

### 3.1 性能需求（移动端优化）

**响应时间**
- API 响应时间：< 300ms（移动端敏感）
- AI 生成时间：< 5 秒（题目解析）
- 图片上传时间：< 2 秒（4G 网络，前端压缩）
- 页面加载时间：< 1.5 秒（首屏）
- 列表滚动帧率：> 50 FPS

**并发性能评估**
```
初期目标（1-3 个月）：
- 日活用户：1000 DAU
- 并发请求：50 QPS
- 数据库连接：20 个
- 存储需求：10GB/月

中期目标（3-12 个月）：
- 日活用户：10000 DAU
- 并发请求：200 QPS
- 数据库连接：50 个
- 存储需求：100GB/月

性能瓶颈预估：
- AI 调用延迟：硅基流动 API 响应时间
- 图片上传：带宽限制
- 数据库：复杂查询和写入
```

**移动端性能优化**
- 列表虚拟化（FlashList）
- 图片懒加载和缓存（expo-image）
- 请求防抖和节流
- 离线缓存策略
- 骨架屏加载

### 3.2 可靠性需求

**可用性**
- 服务可用性：> 99.5%
- 数据持久性：> 99.99%
- 故障恢复时间：< 30 分钟
- 移动端弱网适配：支持 2G/3G/4G/WiFi 自动切换

**数据一致性**
- 用户数据强一致性
- 学习记录最终一致性（移动端可接受短暂不一致）
- 缓存数据最终一致性
- 离线数据本地存储，网络恢复后同步

### 3.3 安全性需求

**认证安全**
- 手机号 + 验证码登录
- JWT Token 认证（移动端本地安全存储）
- Token 自动刷新机制
- 支持多设备登录（最多 3 台）

**数据安全**
- 传输加密（HTTPS/TLS 1.3）
- 敏感数据加密存储（AsyncStorage 加密）
- 防止 SQL 注入
- 图片上传前本地压缩（保护隐私）

**接口安全**
- API 限流（防刷）
- 防重放攻击（Nonce + Timestamp）
- 参数验证（Zod 校验）
- 文件类型校验（防恶意文件）

**移动端特有安全**
- Root/越狱检测（可选）
- 防截屏（敏感页面）
- 本地数据加密
- 生物识别支持（Face ID/Touch ID）

## 7. 技术选型确认

### 7.1 前端技术栈
- **框架**: React Native + TypeScript
- **开发工具**: Expo
- **状态管理**: Zustand（轻量、简单）
- **图片处理**: expo-image
- **列表优化**: FlashList
- **导航**: React Navigation（Native Stack）

### 7.2 BFF 层（Node.js）
- **框架**: Fastify（高性能、TypeScript 友好）
- **认证**: JWT
- **验证**: Zod
- **日志**: Pino

### 7.3 后端技术栈
- **API 框架**: FastAPI
- **Agent 框架**: LangChain
- **主数据库**: PostgreSQL
- **缓存/会话**: Redis
- **文件存储**: 本地存储（后期扩展云存储）

### 7.4 AI 模型
- **主模型**: 硅基流动 THUDM/GLM-4.1V-9B-Thinking
  - 视觉识别（OCR）
  - 题目解析
  - 类似题生成
  - 课件解析
  - 笔记生成

### 7.5 文件解析服务推荐
根据 Skills 调研，推荐以下方案：

**PDF 解析**
- `pdf-parse` (Node.js) - 轻量级 PDF 文本提取
- `PyMuPDF` (Python) - 功能强大，支持图片和文本

**PPT 解析**
- `pptx-parser` (Node.js) - 解析 PPTX 文件
- `python-pptx` (Python) - Python 库，功能全面

**Word 解析**
- `mammoth` (Node.js) - 提取 Word 文本内容
- `python-docx` (Python) - Python 官方库

**Markdown 解析**
- `marked` (Node.js) - 快速 Markdown 解析
- 直接文本处理（Python）

**推荐方案**: 使用 Python 后端统一处理文件解析，利用丰富的文档处理库
